# 创建 CCF CSP 认证真题文件

当用户需要为某一次 CCF CSP 认证创建真题文件时，按照以下步骤操作。用户只需提供**认证次数**（如"第38次"），工作流即可自动完成所有操作。

## 前置信息收集

1. 用户提供**认证次数**（如：第39次、第38次等）。

2. 根据认证次数，自动在 `d:\竞赛真题\CCF CSP认证\题目说明\` 文件夹中查找对应的 `.html` 文件。

   **文件命名格式**：`第{XX}次{YYYY}年{MM}月.html`（如 `第38次2025年06月.html`）。

   **查找方法**：使用 `find_by_name` 或 `list_dir` 工具，按 `第{XX}次*` 模式匹配文件名。

3. 如果找到对应的 `.html` 文件，使用 `view_file` 或 `read_url_content` 工具读取文件内容，然后从中**自动解析**以下信息：

   **HTML 结构说明**：
   - 每道题用 `<div class="problem">` 包裹
   - 题目标题在 `<h2 class="problem-header">` 中，格式如 `第 1 题：正态分布（100 分）`
   - 时间/空间限制在 `<p class="limits">` 中，格式如 `时间限制：1000 ms　|　空间限制：524288 KB`
   - 题目完整描述在 `<div class="question-body">` 中，内含 `<h2>题目描述</h2>`、`<h2>输入格式</h2>`、`<h2>输出格式</h2>` 等子标题

   **需要提取的信息**：
   - **题号**：从标题中提取（如"第 1 题"→ 1）
   - **中文题名**：从标题中提取（如"正态分布"）
   - **时间限制**：从 limits 段落中提取并转换为秒（如 `1000 ms` → `1.0秒`）
   - **内存限制**：从 limits 段落中提取并转换为 MiB（如 `524288 KB` → `512 MiB`）
   - **题目描述**、**输入格式**、**输出格式**、**样例输入/输出**、**数据范围**：从 question-body 中提取纯文本内容（去除 HTML 标签），注意去除其中的图片和公式标签
   - **英文标识**：HTML 文件中**不包含**英文标识信息，需要根据中文题名自行翻译/推断一个简短的英文标识（通常是一到两个英文单词，如：正态分布→normal，机器人复健指南→robot，消息解码→decode，月票发行→ticket，博物馆→museum）

4. 如果**找不到**对应的 `.html` 文件，则退回到手动模式，向用户确认：
   - 每道题的名称和英文标识
   - 是否需要将完整题面写入注释

## 创建目录和文件

// turbo-all

5. 在 `d:\竞赛真题\CCF CSP认证\` 下创建对应的认证目录。

   **目录命名规则**：使用 `{排序前缀}_第{XX}次` 格式，排序前缀为递增整数，**认证次数越高前缀越小**，使其在按名称排序时显示在前面。
   
   确定前缀的方法：
   - 查看现有目录的前缀编号
   - 如果新认证次数比所有现有的都高，则使用最小现有前缀 - 1（如现有最小为 1，则新目录为 0）
   - 如果新认证次数比所有现有的都低，则使用最大现有前缀 + 1
   - 如果在中间，则需要对现有目录重新编号以腾出位置
   
   示例：`1_第40次`、`2_第39次`、`3_第38次`

6. 在该目录下为每道题创建 `.cpp` 文件，文件命名规则：
   ```
   {题号}_{中文题名}_{英文标识}.cpp
   ```
   例如：`1_正态分布_normal.cpp`、`2_机器人复健指南_robot.cpp`

7. 每个 `.cpp` 文件的内容模板如下：

   **如果成功从 HTML 文件解析到题面**，将题面作为注释写入（包含样例输入输出）：
   ```cpp
   /*
    * {中文题名} ({英文标识})
    * CCF CSP 第{XX}次认证 真题{题号}
    * 时间限制: {X.X}秒  内存限制: {XXX} MiB
    *
    * 题目描述：
    * {题目描述}
    *
    * 输入格式：
    * {输入格式说明}
    *
    * 输出格式：
    * {输出格式说明}
    *
    * 样例输入：
    * {样例输入数据，去掉行号}
    *
    * 样例输出：
    * {样例输出数据，去掉行号}
    *
    * 数据范围：
    * {数据范围说明}
    */

   #include <iostream>
   using namespace std;
   int main() {

     return 0;
   }
   ```

   如果有多组样例，则用 `样例输入1/样例输出1`、`样例输入2/样例输出2` 区分。

   **如果未能解析到题面**，使用简化版本：
   ```cpp
   /*
    * {中文题名} ({英文标识})
    * CCF CSP 第{XX}次认证 真题{题号}
    */

   #include <iostream>
   using namespace std;
   int main() {

     return 0;
   }
   ```

8. 文件创建完成后，列出最终的目录结构确认无误。

## 注意事项

- 所有 `.cpp` 文件只包含框架代码（`#include` + `main` 函数），不包含任何解题代码，具体代码由用户自己编写。
- 文件名中的中文题名要与原题保持一致。
- 样例数据直接写在 `.cpp` 文件的注释块中，**不要**创建单独的测试数据目录或文件。
- 题面信息优先从本地 `d:\竞赛真题\CCF CSP认证\题目说明\` 下的 `.html` 文件中自动读取和解析，无需用户手动提供。如果对应的 `.html` 文件不存在，再请用户提供题面内容或题目名称。
- **英文标识**不在 HTML 文件中，需根据中文题名自行推断合理的英文缩写（通常为一个名词或动词，全小写）。
- CCF CSP 认证通常为5道编程题，每题100分，总分500分，难度依次递增。
- 目录使用排序前缀确保认证次数高的排在前面（工作区 `explorer.sortOrder` 设置为 `"default"` 按名称排序）。
- 创建的 `.cpp` 文件可直接通过右上角 ▶ 按钮或 `F6` 编译运行（由 C/C++ Compile Run 扩展处理，exe 输出到源文件同目录），无需手动配置编译命令。
- 从 HTML 提取文本时，注意将公式标签（含 `data-value` 属性的 `<span>` 标签）转换为可读的纯文本形式，忽略图片（`<img>` 标签）。
